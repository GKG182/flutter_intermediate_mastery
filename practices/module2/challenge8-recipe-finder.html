Recipe Finder App

Difficulty: Medium Module: API Integration Concepts: HTTP GET, JSON Parsing, Search, Detail View Est. Time: 45-60 min 🍳 The Meal DB API

📋 Challenge Description

Build a Recipe Finder App that allows users to search for recipes by name, browse recipes by category, and view detailed recipe information including ingredients and instructions. Use the The Meal DB API to fetch recipe data.

🌐 API Endpoints

Search by Name

https://www.themealdb.com/api/json/v1/1/search.php?s={query}

List Categories

https://www.themealdb.com/api/json/v1/1/categories.php

Filter by Category

https://www.themealdb.com/api/json/v1/1/filter.php?c={category}

Lookup by ID

https://www.themealdb.com/api/json/v1/1/lookup.php?i={id}

Returns recipe details including ingredients, instructions, and images.

💡 Note: No API key required! Free to use for learning.

✅ Requirements

  • Search for recipes by name
  • Browse recipes by category
  • Display recipe list with images and names
  • Tap a recipe to see details (ingredients, instructions)
  • Show loading indicator while searching
  • Handle empty results and errors
  • Use http package for API calls
  • Use StateNotifier for state management

📚 Resources

  • The Meal DB API
  • http package
  • cached_network_image
Need a hint? Click here

💡 Hint: Create three models: Recipe (list view), RecipeDetail (full details), and Category . Use StateNotifier to manage search results, selected category, and selected recipe. For ingredients, the API returns them as strIngredient1 through strIngredient20 .

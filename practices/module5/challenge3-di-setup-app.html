Dependency Injection Setup App

Difficulty: Medium Module: Clean Architecture Concepts: Dependency Injection, get_it, Modular Architecture, Environment Config Est. Time: 35-45 min 🏗️ DI

📋 Challenge Description

Build a Dependency Injection Setup App that demonstrates proper use of get_it for dependency injection with modular architecture and environment-based configurations.

✅ Requirements

  • Use get_it for dependency injection
  • Create separate modules for different app features
  • Implement environment-based configuration (dev, staging, prod)
  • Use factory and singleton registrations appropriately
  • Implement lazy loading for non-critical dependencies
  • Create abstract interfaces with multiple implementations
  • Use environment variables for API keys and URLs
  • Show dependency injection in action with a simple UI
  • Implement testing mocks using get_it
  • Handle service registration with proper order

📁 Module Structure

  • lib/
  • ├── core/
  • │ ├── di/ (injection.dart, modules/)
  • │ ├── config/ (environment.dart)
  • │ └── network/ (api_client.dart)
  • ├── features/
  • │ ├── auth/ (data/, domain/, presentation/)
  • │ └── dashboard/ (data/, domain/, presentation/)
  • └── main.dart

📚 Resources

  • get_it Package
  • flutter_dotenv
  • injectable (Code Generation)
  • mockito for testing
Need a hint? Click here

💡 Hint: Create separate modules for each feature (e.g., AuthModule, DashboardModule). Use registerFactory() for transient dependencies and registerSingleton() for shared services. Create an Environment class with dev , staging , and prod configs. Use registerLazySingleton() for services that shouldn't be created at startup.

Counter Evolution System

Difficulty: Easy Module: State Management Concepts: StateProvider, Riverpod Est. Time: 30-45 min

📋 Challenge Description

Build a Counter Evolution System — a multi-counter app that demonstrates the power of Riverpod's StateProvider. Users should be able to create multiple independent counters, increment/decrement each one, reset individual counters, and see a global total that updates automatically.

✅ Requirements

  • Users can add new counters with a "+" button
  • Each counter has increment (+) and decrement (-) buttons
  • Each counter has a reset button to set it back to 0
  • Display a global total that sums all counters
  • Counters should be removable with a delete button
  • Use StateProvider from Riverpod for state management
  • Clean, responsive UI with proper styling

📚 Resources

  • Riverpod Documentation
  • Flutter Interactivity
  • flutter_riverpod package
Need a hint? Click here

💡 Hint: Use a List<int> to store all counter values. Each counter is an index in the list. Use StateProvider<List<int>> to manage the list. For the global total, use a Provider that watches the counter list and calculates the sum using .fold() .

2. Counter Provider (lib/providers/counter_provider.dart)

lib/providers/counter_provider.dart import 'package:flutter_riverpod/flutter_riverpod.dart' ; // StateProvider for the list of counters final countersProvider = StateProvider<List< int >>((ref) { return [ 0 ]; // Start with one counter at 0 }); // Provider for the global total (auto-updates) final totalProvider = Provider< int >((ref) { final counters = ref. watch (countersProvider); return counters. fold ( 0 , (sum, count) => sum + count); });

✅ Key Concepts Demonstrated

  • StateProvider – Managing a list of integers
  • Provider – Derived state (global total)
  • Immutability – Creating new lists when updating state
  • ConsumerWidget – Watching providers in the UI
  • ref.read() – Updating state without rebuilding

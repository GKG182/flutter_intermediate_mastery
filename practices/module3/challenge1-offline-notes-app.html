Offline Notes App

Difficulty: Medium Module: Local Storage Concepts: Hive, CRUD Operations, Riverpod Est. Time: 30-40 min 📦 Hive

📋 Challenge Description

Build an Offline Notes App that allows users to create, read, update, and delete notes. All notes should be stored locally on the device using Hive as the local database.

✅ Requirements

  • Create a Note model with id , title , content , timestamp
  • Implement CRUD operations (Create, Read, Update, Delete)
  • Use Hive for local storage with custom adapter
  • Use Riverpod for state management
  • Show a list of notes on the home screen
  • Navigate to add/edit note screen
  • Add delete functionality (swipe or button)
  • Display timestamps for each note
  • Handle empty state when no notes exist

📦 Hive Setup

Hive is a lightweight, fast key-value database for Flutter.

flutter pub add hive_flutter hive_generator build_runner
💡 Note: Hive runs entirely on the device - no internet required!

📚 Resources

  • Hive Package

Need a hint? Click here

💡 Hint: Create a Note class with @HiveType annotation and @HiveField for each field. Use Hive.box<Note>('notes') to access the database. Wrap your app with HiveProvider . For Riverpod, create a StateNotifier that manages the list of notes and provides CRUD methods.

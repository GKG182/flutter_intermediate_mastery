Expense Tracker Pro

Difficulty: Medium Module: Local Storage Concepts: SQLite, CRUD, Query Filtering, Charts Est. Time: 40-50 min 🗄️ SQLite

📋 Challenge Description

Build an Expense Tracker Pro app that allows users to track their income and expenses. All data should be stored locally using SQLite with the sqflite package.

✅ Requirements

  • Create an Expense model with id , title , amount , category , type (income/expense), date
  • Implement CRUD operations using SQLite
  • Use sqflite package for database operations
  • Use Riverpod for state management
  • Show a summary dashboard with:
    • Total income
    • Total expenses
    • Net balance
  • Display a list of transactions sorted by date
  • Add filter by category functionality
  • Add filter by date range (optional)
  • Include a chart showing spending by category
  • Handle empty state when no transactions exist

🗄️ SQLite Setup

sqflite is a SQLite plugin for Flutter with full SQL support.

flutter pub add sqflite path_provider
💡 Note: SQLite provides powerful querying capabilities for filtering and aggregating data.

📚 Resources

  • sqflite Package
  • path_provider
  • Flutter SQLite Cookbook
  • fl_chart for charts
Need a hint? Click here

💡 Hint: Create a database helper class with methods for insert , update , delete , and query . Use FutureProvider for the list of transactions and summary data. For charts, use fl_chart package to display a pie chart of spending by category.

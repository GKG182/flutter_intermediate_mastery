Live Location Tracker

Difficulty: Hard Module: Supabase Concepts: Geolocation, Google Maps, Supabase Realtime, Privacy Controls Est. Time: 60-75 min 🗄️ Supabase

📋 Challenge Description

Build a Live Location Tracker app where users can share their real-time location with friends and see live updates on a map. This challenge combines Google Maps integration, Supabase Realtime, and privacy controls.

✅ Requirements

  • Use Supabase Auth for user authentication
  • Use Google Maps Flutter for map display
  • Use Geolocator for device location
  • Create a locations table with user_id , latitude , longitude , timestamp , is_sharing
  • Enable Realtime on the locations table
  • Show all active users on the map with custom markers
  • Update locations in real-time as users move
  • Add privacy controls to toggle location sharing on/off
  • Show user avatars on markers with online status
  • Display distance between users
  • Add friend list showing who is online and sharing
  • Handle permissions gracefully

🗄️ Supabase Setup

Tables needed:

  • locations (user_id, latitude, longitude, timestamp, is_sharing, last_updated)
  • friends (user_id, friend_id, status, created_at)
💡 Note: Enable Realtime on the locations table and set up RLS policies.

📚 Resources

  • Google Maps Flutter
  • Geolocator
  • Supabase Realtime Docs
  • Row Level Security
Need a hint? Click here

💡 Hint: Use Geolocator.getPositionStream() to listen to location updates. Use supabase.from('locations').upsert() to update the user's location. Create a marker for each user with a custom icon showing their avatar. Use computeDistanceBetween() from the maps package to calculate distances. Handle location permissions with Geolocator.requestPermission() .

2. SQL Schema (supabase/migrations/location_tracker.sql)

supabase/migrations/location_tracker.sql -- Create locations table CREATE TABLE locations ( user_id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE, latitude DOUBLE PRECISION NOT NULL, longitude DOUBLE PRECISION NOT NULL, timestamp TIMESTAMPTZ DEFAULT NOW(), is_sharing BOOLEAN DEFAULT true, last_updated TIMESTAMPTZ DEFAULT NOW() ); -- Create friends table CREATE TABLE friends ( user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE, friend_id UUID REFERENCES auth.users(id) ON DELETE CASCADE, status TEXT DEFAULT 'pending', created_at TIMESTAMPTZ DEFAULT NOW(), PRIMARY KEY (user_id, friend_id) ); -- Enable Row Level Security ALTER TABLE locations ENABLE ROW LEVEL SECURITY; ALTER TABLE friends ENABLE ROW LEVEL SECURITY; -- Locations policies CREATE POLICY "Users can see all locations" ON locations FOR SELECT USING (true); CREATE POLICY "Users can update their own location" ON locations FOR INSERT WITH CHECK (auth.uid() = user_id); CREATE POLICY "Users can update their own location" ON locations FOR UPDATE USING (auth.uid() = user_id); -- Friends policies CREATE POLICY "Users can see their friends" ON friends FOR SELECT USING (auth.uid() = user_id OR auth.uid() = friend_id); CREATE POLICY "Users can add friends" ON friends FOR INSERT WITH CHECK (auth.uid() = user_id); -- Enable Realtime ALTER TABLE locations REPLICA IDENTITY FULL; ALTER PUBLICATION supabase_realtime ADD TABLE locations;

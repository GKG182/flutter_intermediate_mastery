Task Management System

Difficulty: Hard Module: Supabase Concepts: Supabase Realtime, Complex Relations, Storage, RLS Est. Time: 60-75 min 🗄️ Supabase

📋 Challenge Description

Build a Collaborative Task Management System where users can create projects, manage tasks, assign team members, and track progress in real-time. This challenge combines multiple Supabase features including Auth, Realtime, Storage, and complex database relationships.

✅ Requirements

  • Use Supabase Auth for user authentication
  • Create projects with name, description, and owner
  • Create tasks with title, description, status, priority, due date, and assigned user
  • Add comments to tasks with real-time updates
  • Upload file attachments to tasks using Supabase Storage
  • Implement Real-time subscriptions for tasks and comments
  • Show project dashboard with task statistics
  • Add task filtering by status, priority, and assignee
  • Implement Row Level Security policies
  • Handle loading states and errors gracefully

🗄️ Database Schema

Tables needed:

  • projects (id, name, description, owner_id, created_at)
  • tasks (id, project_id, title, description, status, priority, assigned_to, due_date, created_at)
  • task_comments (id, task_id, user_id, content, created_at)
  • task_attachments (id, task_id, file_url, file_name, uploaded_by, created_at)
  • project_members (project_id, user_id, role)
💡 Note: Enable Realtime on all tables and set up proper RLS policies.

📚 Resources

  • Supabase Flutter Package
  • Supabase Realtime Docs
  • Supabase Storage Docs
  • Row Level Security

2. SQL Schema (supabase/migrations/task_management.sql)

supabase/migrations/task_management.sql -- Create tables CREATE TABLE projects ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, name TEXT NOT NULL, description TEXT, owner_id UUID NOT NULL REFERENCES auth.users(id), created_at TIMESTAMPTZ DEFAULT NOW() ); CREATE TABLE project_members ( project_id UUID REFERENCES projects(id) ON DELETE CASCADE, user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE, role TEXT DEFAULT 'member', joined_at TIMESTAMPTZ DEFAULT NOW(), PRIMARY KEY (project_id, user_id) ); CREATE TABLE tasks ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, project_id UUID REFERENCES projects(id) ON DELETE CASCADE, title TEXT NOT NULL, description TEXT, status TEXT DEFAULT 'todo', priority TEXT DEFAULT 'medium', assigned_to UUID REFERENCES auth.users(id), due_date TIMESTAMPTZ, created_by UUID REFERENCES auth.users(id), created_at TIMESTAMPTZ DEFAULT NOW(), updated_at TIMESTAMPTZ DEFAULT NOW() ); CREATE TABLE task_comments ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, task_id UUID REFERENCES tasks(id) ON DELETE CASCADE, user_id UUID REFERENCES auth.users(id), content TEXT NOT NULL, created_at TIMESTAMPTZ DEFAULT NOW() ); CREATE TABLE task_attachments ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, task_id UUID REFERENCES tasks(id) ON DELETE CASCADE, file_url TEXT NOT NULL, file_name TEXT NOT NULL, file_size INTEGER, uploaded_by UUID REFERENCES auth.users(id), created_at TIMESTAMPTZ DEFAULT NOW() ); -- Enable Row Level Security ALTER TABLE projects ENABLE ROW LEVEL SECURITY; ALTER TABLE project_members ENABLE ROW LEVEL SECURITY; ALTER TABLE tasks ENABLE ROW LEVEL SECURITY; ALTER TABLE task_comments ENABLE ROW LEVEL SECURITY; ALTER TABLE task_attachments ENABLE ROW LEVEL SECURITY; -- Projects: Users can see projects they own or are members of CREATE POLICY "Users can view their projects" ON projects FOR SELECT USING ( auth.uid() = owner_id OR auth.uid() IN (SELECT user_id FROM project_members WHERE project_id = projects.id) ); CREATE POLICY "Users can insert their own projects" ON projects FOR INSERT WITH CHECK (auth.uid() = owner_id); CREATE POLICY "Users can update their own projects" ON projects FOR UPDATE USING (auth.uid() = owner_id); -- Tasks: Users can see tasks in their projects CREATE POLICY "Users can view project tasks" ON tasks FOR SELECT USING ( auth.uid() IN ( SELECT user_id FROM project_members WHERE project_id = tasks.project_id UNION SELECT owner_id FROM projects WHERE id = tasks.project_id ) ); CREATE POLICY "Users can insert tasks in their projects" ON tasks FOR INSERT WITH CHECK ( auth.uid() IN ( SELECT user_id FROM project_members WHERE project_id = tasks.project_id UNION SELECT owner_id FROM projects WHERE id = tasks.project_id ) ); CREATE POLICY "Users can update tasks in their projects" ON tasks FOR UPDATE USING ( auth.uid() IN ( SELECT user_id FROM project_members WHERE project_id = tasks.project_id UNION SELECT owner_id FROM projects WHERE id = tasks.project_id ) ); -- Comments: Similar policies CREATE POLICY "Users can view task comments" ON task_comments FOR SELECT USING ( auth.uid() IN ( SELECT user_id FROM project_members pm JOIN tasks t ON t.project_id = pm.project_id WHERE t.id = task_comments.task_id UNION SELECT owner_id FROM projects p JOIN tasks t ON t.project_id = p.id WHERE t.id = task_comments.task_id ) ); CREATE POLICY "Users can insert comments on their tasks" ON task_comments FOR INSERT WITH CHECK (auth.uid() = user_id); -- Enable Realtime ALTER TABLE projects REPLICA IDENTITY FULL; ALTER TABLE tasks REPLICA IDENTITY FULL; ALTER TABLE task_comments REPLICA IDENTITY FULL; ALTER TABLE task_attachments REPLICA IDENTITY FULL; -- Add tables to Realtime publication ALTER PUBLICATION supabase_realtime ADD TABLE projects; ALTER PUBLICATION supabase_realtime ADD TABLE tasks; ALTER PUBLICATION supabase_realtime ADD TABLE task_comments; ALTER PUBLICATION supabase_realtime ADD TABLE task_attachments; -- Create storage bucket for attachments INSERT INTO storage.buckets (id, name, public) VALUES ('task_attachments', 'task_attachments', true); -- Storage policy: Users can upload attachments for tasks they have access to CREATE POLICY "Users can upload task attachments" ON storage.objects FOR INSERT WITH CHECK ( bucket_id = 'task_attachments' AND auth.uid() IN ( SELECT user_id FROM project_members pm JOIN tasks t ON t.project_id = pm.project_id WHERE t.id = (SELECT (regexp_match(name, '^([^/]+)'))[1]::UUID) UNION SELECT owner_id FROM projects p JOIN tasks t ON t.project_id = p.id WHERE t.id = (SELECT (regexp_match(name, '^([^/]+)'))[1]::UUID) ) );

4. Providers (lib/providers/)

lib/providers/task_provider.dart import 'dart:async'; import 'package:flutter_riverpod/flutter_riverpod.dart'; import 'package:supabase_flutter/supabase_flutter.dart'; import '../models/task.dart'; import '../models/comment.dart'; import '../models/attachment.dart'; import '../models/project.dart'; final supabase = Supabase.instance.client; // Projects final projectsProvider = FutureProvider<List<Project>>((ref) async { final response = await supabase .from('projects') .select('*') .order('created_at', ascending: false); return (response as List) .map((json) => Project.fromJson(json)) .toList(); }); // Tasks with real-time updates final tasksProvider = StreamProvider<List<Task>>((ref) { return _getTasksStream(); }); Stream<List<Task>> _getTasksStream() async* { // Load initial tasks final initialResponse = await supabase .from('tasks') .select('*') .order('created_at', ascending: false); List<Task> tasks = (initialResponse as List) .map((json) => Task.fromJson(json)) .toList(); final controller = StreamController<List<Task>>(); controller.add(tasks); // Create channel for real-time updates final channel = supabase.channel('public:tasks'); channel.on( RealtimeListenTypes.postgresChanges, ChannelFilter( event: '*', schema: 'public', table: 'tasks', ), (payload, [ref]) { final event = payload['event_type']; if (event == 'INSERT') { final newTask = Task.fromJson(payload['new'] as Map<String, dynamic>); tasks = [newTask, ...tasks]; controller.add(tasks); } else if (event == 'UPDATE') { final updatedTask = Task.fromJson(payload['new'] as Map<String, dynamic>); final index = tasks.indexWhere((t) => t.id == updatedTask.id); if (index != -1) { tasks[index] = updatedTask; controller.add(tasks); } } else if (event == 'DELETE') { final deletedId = payload['old']['id']; tasks.removeWhere((t) => t.id == deletedId); controller.add(tasks); } }, ); await channel.subscribe(); await for (final _ in controller.stream) { yield tasks; } }); // Task operations final taskNotifierProvider = Provider<TaskNotifier>((ref) { return TaskNotifier(); }); class TaskNotifier { Future<void> createTask(Task task) async { await supabase .from('tasks') .insert(task.toInsertJson()); } Future<void> updateTask(Task task) async { await supabase .from('tasks') .update(task.toJson()) .match({'id': task.id}); } Future<void> deleteTask(String taskId) async { await supabase .from('tasks') .delete() .match({'id': taskId}); } Future<void> updateTaskStatus(String taskId, String status) async { await supabase .from('tasks') .update({'status': status, 'updated_at': DateTime.now().toIso8601String()}) .match({'id': taskId}); } } // Comments final commentsProvider = StreamProvider.family<List<Comment>, String>((ref, taskId) { return _getCommentsStream(taskId); }); Stream<List<Comment>> _getCommentsStream(String taskId) async* { final initialResponse = await supabase .from('task_comments') .select('*') .eq('task_id', taskId) .order('created_at', ascending: false); List<Comment> comments = (initialResponse as List) .map((json) => Comment.fromJson(json)) .toList(); final controller = StreamController<List<Comment>>(); controller.add(comments); final channel = supabase.channel('public:task_comments:$taskId'); channel.on( RealtimeListenTypes.postgresChanges, ChannelFilter( event: 'INSERT', schema: 'public', table: 'task_comments', filter: 'task_id=eq.$taskId', ), (payload, [ref]) { final newComment = Comment.fromJson(payload['new'] as Map<String, dynamic>); comments = [newComment, ...comments]; controller.add(comments); }, ); await channel.subscribe(); await for (final _ in controller.stream) { yield comments; } }); final commentNotifierProvider = Provider<CommentNotifier>((ref) { return CommentNotifier(); }); class CommentNotifier { Future<void> addComment(Comment comment) async { await supabase .from('task_comments') .insert(comment.toInsertJson()); } } // Attachments final attachmentsProvider = FutureProvider.family<List<Attachment>, String>((ref, taskId) async { final response = await supabase .from('task_attachments') .select('*') .eq('task_id', taskId) .order('created_at', ascending: false); return (response as List) .map((json) => Attachment.fromJson(json)) .toList(); }); final attachmentNotifierProvider = Provider<AttachmentNotifier>((ref) { return AttachmentNotifier(); }); class AttachmentNotifier { Future<String> uploadFile({ required String taskId, required String filePath, required String fileName, }) async { final file = File(filePath); final bytes = await file.readAsBytes(); final fileSize = bytes.lengthInBytes; final path = '$taskId/${DateTime.now().millisecondsSinceEpoch}_$fileName'; await supabase.storage .from('task_attachments') .upload(path, bytes); final url = supabase.storage .from('task_attachments') .getPublicUrl(path); final user = supabase.auth.currentUser; if (user == null) throw Exception('User not authenticated'); await supabase .from('task_attachments') .insert({ 'task_id': taskId, 'file_url': url, 'file_name': fileName, 'file_size': fileSize, 'uploaded_by': user.id, }); return url; } Future<void> deleteAttachment(String attachmentId) async { // Get the attachment first to get the file path final response = await supabase .from('task_attachments') .select('file_url') .eq('id', attachmentId) .single(); final fileUrl = response['file_url'] as String; // Extract path from URL (remove base URL) final path = fileUrl.split('/task_attachments/')[1]; // Delete from storage await supabase.storage .from('task_attachments') .remove([path]); // Delete from database await supabase .from('task_attachments') .delete() .match({'id': attachmentId}); } }

✅ Key Concepts Demonstrated

  • Supabase Realtime – Live updates for tasks and comments across multiple tables
  • Complex Database Relations – Foreign keys, joins, and cascading deletes
  • Row Level Security – Granular access control for projects, tasks, and comments
  • Supabase Storage – File uploads with public/private buckets
  • StreamProvider – Real-time data streams with Riverpod
  • Multi-table Subscriptions – Separate channels for different data types

🛒 Project 3

Shopping Cart System

Build a complete shopping cart with product catalog, cart management, and checkout flow using Riverpod StateNotifierProvider.

Riverpod
📋 Requirements

What You'll Build

  • Product catalog – Display a list of products with name, price, and image
  • Add to cart – Add products to the shopping cart with quantity selection
  • Cart management – View cart items, update quantities, and remove items
  • Cart totals – Calculate subtotal, tax, and total price
  • Checkout flow – Simple checkout with order summary
  • Stock management – Track product stock and prevent over-purchasing
  • Cart persistence – Cart persists across app sessions (using SharedPreferences)

Learning Objectives

  • Use StateNotifierProvider with complex state
  • Work with immutable state and copyWith
  • Manage lists with add, remove, and update operations
  • Calculate derived state (totals, taxes)
  • Persist state across sessions
  • Build a multi-screen e-commerce flow
📦 Product Data (API Simulation)

Product Catalog (Free API)

  • Using a mock API with sample product data
  • Products include: id, name, price, description, imageUrl, stock
  • Image URLs: Using picsum.photos for placeholder images
  • Data source: Hardcoded product list (simulates an API)

CART NOTIFIER // ========================================== class CartNotifier extends StateNotifier<CartState> { CartNotifier() : super(CartState.initial()); // Add item to cart void addItem(Product product, {int quantity = 1}) { final existingIndex = state.items.indexWhere( (item) => item.product.id == product.id, ); List<CartItem> newItems; if (existingIndex != -1) { // Product exists, update quantity final existingItem = state.items[existingIndex]; final newQuantity = existingItem.quantity + quantity; // Cap at stock final finalQuantity = newQuantity > product.stock ? product.stock : newQuantity; newItems = List.from(state.items); newItems[existingIndex] = CartItem( product: product, quantity: finalQuantity, ); } else { // New product final finalQuantity = quantity > product.stock ? product.stock : quantity; newItems = [...state.items, CartItem(product: product, quantity: finalQuantity)]; } state = state.copyWith(items: newItems); _saveCart(); } // Remove item from cart void removeItem(String productId) { state = state.copyWith( items: state.items.where((item) => item.product.id != productId).toList(), ); _saveCart(); } // Update quantity void updateQuantity(String productId, int newQuantity) { if (newQuantity <= 0) { removeItem(productId); return; } final itemIndex = state.items.indexWhere( (item) => item.product.id == productId, ); if (itemIndex == -1) return; final product = state.items[itemIndex].product; final finalQuantity = newQuantity > product.stock ? product.stock : newQuantity; List<CartItem> newItems = List.from(state.items); newItems[itemIndex] = CartItem( product: product, quantity: finalQuantity, ); state = state.copyWith(items: newItems); _saveCart(); } // Clear cart void clearCart() { state = CartState.initial(); _saveCart(); } // Save cart to SharedPreferences void _saveCart() { // In a real app, save to SharedPreferences // For this demo, we're just printing print('Cart saved: ${state.items.length} items'); } // Load cart from SharedPreferences void loadCart() { // In a real app, load from SharedPreferences print('Cart loaded'); } } // ========================================== // 5. 📖 How This Solution Works

Step-by-Step Explanation

  • 1. Models – Product and CartItem define the data structures.
  • 2. Product Data – A mock product list simulates an API with 6 sample products.
  • 3. Cart State – CartState holds cart items and computes totals (subtotal, tax, total).
  • 4. Cart Notifier – CartNotifier manages add, remove, update quantity, and clear operations.
  • 5. Stock Management – Quantity updates are capped by available stock.
  • 6. UI – Two tabs: Shop (product grid) and Cart (item list with totals).
  • 7. Checkout – Order confirmation dialog with cart clearing.
Immutable State
All state updates create new lists and objects using copyWith, ensuring predictable changes.
Derived State
Totals, taxes, and item counts are computed from the cart state, not stored separately.
Stock Validation
Quantity updates respect product stock limits to prevent over-purchasing.
Cart Persistence
The _saveCart() method prepares data for SharedPreferences (implementation shown).

🎯 Key Takeaway

This project demonstrates a complete e-commerce shopping cart using StateNotifierProvider. It showcases complex state management with immutable data, computed properties, validation, and a full user flow from browsing to checkout. The pattern is production-ready and scalable.

📱 Project 1

Counter App Pro

Build a professional counter app with increment, decrement, reset, and step control using Riverpod state management.

Flutter Docs – Interactivity Riverpod
📋 Requirements

What You'll Build

  • A counter app with increment and decrement buttons
  • A reset button to set the counter to zero
  • A step size controller (increase/decrease by 1, 5, or 10)
  • Min/Max limits – counter cannot go below 0 or above 100
  • History tracking – show the last 5 values
  • A visual progress bar showing the counter value

Learning Objectives

  • Understand StatefulWidget and setState()
  • Learn StateProvider in Riverpod
  • Practice reading and updating state
  • Handle user input and validation
  • Build a clean, responsive UI

NOTIFIER CLASS WITH BUSINESS LOGIC // ========================================== class CounterNotifier { final WidgetRef ref; CounterNotifier(this.ref); // Constants static const int minValue = 0; static const int maxValue = 100; static const int maxHistory = 5; // Increment void increment() { final current = ref.read(counterProvider); final step = ref.read(stepProvider); if (current + step <= maxValue) { final newValue = current + step; ref.read(counterProvider.notifier).state = newValue; _addToHistory(newValue); } } // Decrement void decrement() { final current = ref.read(counterProvider); final step = ref.read(stepProvider); if (current - step >= minValue) { final newValue = current - step; ref.read(counterProvider.notifier).state = newValue; _addToHistory(newValue); } } // Reset void reset() { ref.read(counterProvider.notifier).state = 0; _addToHistory(0); } // Set step void setStep(int step) { ref.read(stepProvider.notifier).state = step; } // Add to history void _addToHistory(int value) { final history = ref.read(historyProvider); final newHistory = [value, ...history]; if (newHistory.length > maxHistory) { newHistory.removeLast(); } ref.read(historyProvider.notifier).state = newHistory; } // Get progress percentage double getProgress(int value) { return value / maxValue; } } // ========================================== // 3. 📖 How This Solution Works

Step-by-Step Explanation

  • 1. Providers – Three providers manage the counter value, step size, and history list.
  • 2. CounterNotifier – Contains all business logic: increment, decrement, reset, and history management.
  • 3. Min/Max Limits – Counter is constrained between 0 and 100 with validation.
  • 4. Step Control – Users can change the step size (1, 5, or 10) which affects increment/decrement.
  • 5. History – The last 5 values are tracked and displayed as chips.
  • 6. Progress Bar – Visual indicator showing how close the counter is to the maximum value.
  • 7. UI Layout – Clean, responsive design with a gradient counter display and status indicators.
Provider
Three StateProviders manage counter, step, and history. This keeps the state separate from the UI.
Notifier Pattern
The CounterNotifier class contains all business logic, making it easy to test and maintain.
Immutability
Each state update creates a new value. History is created as a new list with the spread operator.
Validation
Min/max limits are enforced before updating the counter, preventing invalid states.

🎯 Key Takeaway

This project demonstrates how to use StateProvider with a separate notifier class to manage complex state logic. The separation of concerns makes the code testable, maintainable, and easy to extend.

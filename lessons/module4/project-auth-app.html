🔐 Project 1

Authentication App

Build a complete authentication app with Supabase — signup, login, logout, password reset, session management, and profile creation.

Supabase Auth supabase_flutter
📋 Requirements

What You'll Build

  • A complete authentication app with Supabase Auth
  • Sign Up – Create new user accounts with email and password
  • Log In – Secure login with email and password
  • Log Out – End user sessions securely
  • Password Reset – Forgot password flow with email reset links
  • Session Management – Persistent sessions across app restarts
  • Profile Creation – Store user profiles in PostgreSQL
  • Protected Routes – Secure pages that require authentication
  • Loading States – User feedback during async operations

Learning Objectives

  • Implement Supabase Auth with email/password
  • Manage user sessions with persistent storage
  • Create user profiles in PostgreSQL
  • Handle password reset flows
  • Build protected routes with auth guards
  • Handle loading and error states
🔧 Supabase Setup
SQL – Create Profiles Table -- Create profiles table that extends auth.users CREATE TABLE public.profiles ( id UUID REFERENCES auth.users(id) PRIMARY KEY, username TEXT UNIQUE, full_name TEXT, avatar_url TEXT, bio TEXT, created_at TIMESTAMPTZ DEFAULT NOW(), updated_at TIMESTAMPTZ DEFAULT NOW() ); -- Enable RLS ALTER TABLE public.profiles ENABLE ROW LEVEL SECURITY; -- Policies CREATE POLICY "Users can read their own profile" ON public.profiles FOR SELECT USING (auth.uid() = id); CREATE POLICY "Users can update their own profile" ON public.profiles FOR UPDATE USING (auth.uid() = id) WITH CHECK (auth.uid() = id); CREATE POLICY "Users can insert their own profile" ON public.profiles FOR INSERT WITH CHECK (auth.uid() = id); -- Function to auto-create profile on signup CREATE OR REPLACE FUNCTION public.handle_new_user() RETURNS TRIGGER AS $$ BEGIN INSERT INTO public.profiles (id, username, full_name) VALUES ( NEW.id, COALESCE(NEW.raw_user_meta_data->>'username', NEW.email), COALESCE(NEW.raw_user_meta_data->>'full_name', NEW.email) ); RETURN NEW; END; $$ LANGUAGE plpgsql SECURITY DEFINER; -- Trigger to auto-create profile CREATE TRIGGER on_auth_user_created AFTER INSERT ON auth.users FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();

📖 How This Solution Works

Step-by-Step Explanation

  • 1. Database Setup – Profiles table extends auth.users with RLS policies and auto-creation trigger.
  • 2. Auth Service – Centralized service for all auth operations with error handling.
  • 3. Auth Gate – Checks authentication state and routes to login or home screen.
  • 4. Login Screen – Email/password login with validation and loading states.
  • 5. Sign Up Screen – Account creation with profile data and verification flow.
  • 6. Home Screen – Protected dashboard showing user profile and logout option.
  • 7. Profile Screen – Edit and update user profile information.
  • 8. Reset Password – Password reset flow with email link.
Auth Service
Centralizes all auth operations with user-friendly error messages for common scenarios.
Profile Creation
Auto-creates profiles on signup using PostgreSQL triggers. Manual creation fallback for existing users.
Protected Routes
AuthGate ensures only authenticated users can access the home screen and profile.
Error Handling
User-friendly error messages for invalid credentials, duplicate emails, and network errors.

🎯 Key Takeaway

This project demonstrates a complete authentication system with Supabase Auth. You've learned to handle signup, login, logout, password reset, and profile management . The Auth Gate pattern ensures protected routes, and the error handling provides a smooth user experience. Always use RLS policies to protect user data!

📸 Project 3

Instagram Mini Clone

Build a complete Instagram-inspired app with Supabase — authentication, user profiles, image upload, posts, likes, comments, and followers.

Supabase Auth Supabase Storage Supabase Realtime
📋 Requirements

What You'll Build

  • A complete Instagram-inspired social app with Supabase
  • Authentication – Login/signup with Supabase Auth
  • User Profiles – Profile creation and editing with avatar upload
  • Image Upload – Upload post images to Supabase Storage
  • Posts – Create, view, and delete posts with images
  • Likes – Like and unlike posts
  • Comments – Add and view comments on posts
  • Followers – Follow and unfollow users
  • Feed System – Timeline showing posts from followed users

Learning Objectives

  • Combine Auth, Database, Storage, and Realtime
  • Implement image upload with Supabase Storage
  • Build social features (likes, comments, follows)
  • Create a feed system with PostgreSQL queries
  • Handle complex relationships with foreign keys
🗄️ Database Setup
SQL – Create Instagram Tables -- 1. Profiles table CREATE TABLE public.profiles ( id UUID REFERENCES auth.users(id) PRIMARY KEY, username TEXT UNIQUE NOT NULL, full_name TEXT, bio TEXT, avatar_url TEXT, website TEXT, created_at TIMESTAMPTZ DEFAULT NOW(), updated_at TIMESTAMPTZ DEFAULT NOW() ); -- 2. Posts table CREATE TABLE public.posts ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, user_id UUID REFERENCES public.profiles(id) ON DELETE CASCADE, image_url TEXT NOT NULL, caption TEXT, created_at TIMESTAMPTZ DEFAULT NOW(), updated_at TIMESTAMPTZ DEFAULT NOW() ); -- 3. Likes table CREATE TABLE public.likes ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, post_id UUID REFERENCES public.posts(id) ON DELETE CASCADE, user_id UUID REFERENCES public.profiles(id) ON DELETE CASCADE, created_at TIMESTAMPTZ DEFAULT NOW(), UNIQUE(post_id, user_id) ); -- 4. Comments table CREATE TABLE public.comments ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, post_id UUID REFERENCES public.posts(id) ON DELETE CASCADE, user_id UUID REFERENCES public.profiles(id) ON DELETE CASCADE, content TEXT NOT NULL, created_at TIMESTAMPTZ DEFAULT NOW(), updated_at TIMESTAMPTZ DEFAULT NOW() ); -- 5. Follows table CREATE TABLE public.follows ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, follower_id UUID REFERENCES public.profiles(id) ON DELETE CASCADE, following_id UUID REFERENCES public.profiles(id) ON DELETE CASCADE, created_at TIMESTAMPTZ DEFAULT NOW(), UNIQUE(follower_id, following_id) ); -- 6. Enable RLS ALTER TABLE public.profiles ENABLE ROW LEVEL SECURITY; ALTER TABLE public.posts ENABLE ROW LEVEL SECURITY; ALTER TABLE public.likes ENABLE ROW LEVEL SECURITY; ALTER TABLE public.comments ENABLE ROW LEVEL SECURITY; ALTER TABLE public.follows ENABLE ROW LEVEL SECURITY; -- 7. RLS Policies CREATE POLICY "Users can read all profiles" ON public.profiles FOR SELECT USING (true); CREATE POLICY "Users can update own profile" ON public.profiles FOR UPDATE USING (auth.uid() = id); CREATE POLICY "Users can read all posts" ON public.posts FOR SELECT USING (true); CREATE POLICY "Users can insert own posts" ON public.posts FOR INSERT WITH CHECK (auth.uid() = user_id); CREATE POLICY "Users can update own posts" ON public.posts FOR UPDATE USING (auth.uid() = user_id); CREATE POLICY "Users can delete own posts" ON public.posts FOR DELETE USING (auth.uid() = user_id); CREATE POLICY "Users can read all likes" ON public.likes FOR SELECT USING (true); CREATE POLICY "Users can insert own likes" ON public.likes FOR INSERT WITH CHECK (auth.uid() = user_id); CREATE POLICY "Users can delete own likes" ON public.likes FOR DELETE USING (auth.uid() = user_id); CREATE POLICY "Users can read all comments" ON public.comments FOR SELECT USING (true); CREATE POLICY "Users can insert own comments" ON public.comments FOR INSERT WITH CHECK (auth.uid() = user_id); CREATE POLICY "Users can delete own comments" ON public.comments FOR DELETE USING (auth.uid() = user_id); CREATE POLICY "Users can read all follows" ON public.follows FOR SELECT USING (true); CREATE POLICY "Users can insert own follows" ON public.follows FOR INSERT WITH CHECK (auth.uid() = follower_id); CREATE POLICY "Users can delete own follows" ON public.follows FOR DELETE USING (auth.uid() = follower_id); -- 8. Storage bucket for post images INSERT INTO storage.buckets (id, name, public) VALUES ('posts', 'posts', true); -- 9. Storage policy for uploads CREATE POLICY "Users can upload images" ON storage.objects FOR INSERT TO authenticated WITH CHECK ( bucket_id = 'posts' AND (storage.foldername(name))[1] = auth.uid()::text ); CREATE POLICY "Anyone can view images" ON storage.objects FOR SELECT TO anon USING (bucket_id = 'posts'); -- 10. Auto-create profile on signup CREATE OR REPLACE FUNCTION public.handle_new_user() RETURNS TRIGGER AS $$ BEGIN INSERT INTO public.profiles (id, username, full_name) VALUES ( NEW.id, COALESCE(NEW.raw_user_meta_data->>'username', NEW.email), COALESCE(NEW.raw_user_meta_data->>'full_name', NEW.email) ); RETURN NEW; END; $$ LANGUAGE plpgsql SECURITY DEFINER; CREATE TRIGGER on_auth_user_created AFTER INSERT ON auth.users FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();

📖 How This Solution Works

Step-by-Step Explanation

  • 1. Database Setup – Five tables: profiles, posts, likes, comments, follows with proper RLS policies and storage bucket for images.
  • 2. Instagram Service – Central service for all CRUD operations including posts, likes, comments, follows, and storage.
  • 3. Authentication – Login/signup with Supabase Auth and auto-profile creation on signup.
  • 4. Feed System – Shows posts from followed users using a query with inFilter.
  • 5. Image Upload – Uses Supabase Storage with public bucket for post images.
  • 6. Social Features – Likes, comments, and follows with toggle functionality.
  • 7. Profile System – View and edit profiles with avatar support.
Storage Integration
Post images are uploaded to Supabase Storage with public access, and URLs are stored in the posts table.
Feed Query
The feed uses inFilter to get posts from users the current user follows, with a nested join to profiles.
RLS Security
Policies ensure users can only modify their own data while reading public content like posts and profiles.
Cached Images
Uses cached_network_image for efficient image loading with placeholders and error handling.

This project demonstrates a complete social media app with Supabase combining Auth, Database, Storage, and Realtime. You've learned to implement authentication, user profiles, image upload, posts, likes, comments, and followers. Supabase is a powerful backend for social apps!

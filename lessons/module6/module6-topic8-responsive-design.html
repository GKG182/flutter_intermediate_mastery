Topic 8

Responsive Design

Building Flutter apps that adapt beautifully to any screen size — from watches to desktops.

Adaptive & Responsive Docs General Approach
Responsive vs Adaptive

One of Flutter's primary goals is to allow you to develop apps from a single codebase that look and feel great on any platform. Your app might appear on screens of many different sizes — from a watch, to a foldable phone, to a high-definition monitor.

💡 Two Approaches

Responsive design is about fitting the UI into the space .
Adaptive design is about the UI being usable in the space .

📱 Responsive

  • Adjusts placement of design elements
  • Fits UI into available space
  • Same layout, different sizes
  • Example: Text scales, columns wrap

🔄 Adaptive

  • Selects appropriate layout
  • UI is usable in the space
  • Different layouts for different sizes
  • Example: Bottom nav vs side nav

✅ The Goal

Ideally, you want your app to be both responsive and adaptive — it should fit the space AND be usable in that space.

Breakpoints

Breakpoints are the screen widths at which your UI changes. Material Design provides recommended breakpoints:

Breakpoint Width Range Recommended Layout
Compact < 600 dp Bottom navigation, single column
Medium 600 - 840 dp Navigation rail, two columns
Expanded > 840 dp Side navigation, multiple columns
Defining Breakpoints class Breakpoint { static const double mobile = 600 ; static const double tablet = 840 ; static const double desktop = 1200 ; static bool isMobile (BuildContext context) { return MediaQuery. sizeOf (context).width < mobile; } static bool isTablet (BuildContext context) { final width = MediaQuery. sizeOf (context).width; return width >= mobile && width < desktop; } static bool isDesktop (BuildContext context) { return MediaQuery. sizeOf (context).width >= desktop; } }
Measuring Screen Space

Flutter provides two main ways to determine the size of your display area: MediaQuery and LayoutBuilder .

📐
MediaQuery
Gets the size of the app's entire window
Use when sizing is based on the whole screen
📦
LayoutBuilder
Gets constraints from the parent widget
Use when sizing is based on a specific area
Using MediaQuery class ResponsiveWidget extends StatelessWidget { @override Widget build (BuildContext context) { final size = MediaQuery. sizeOf (context); final width = size.width; final height = size.height; if (width < 600 ) { return MobileLayout(); } else if (width < 1200 ) { return TabletLayout(); } else { return DesktopLayout(); } } }
Using LayoutBuilder class ResponsiveContainer extends StatelessWidget { @override Widget build (BuildContext context) { return LayoutBuilder( builder: (context, constraints) { final maxWidth = constraints.maxWidth; if (maxWidth < 400 ) { return Column( children: [Text( 'Small' ), Text( 'Content' )], ); } else { return Row( children: [Text( 'Large' ), Expanded(child: Text( 'Content' ))], ); } }, ); } }

💡 Which to Use?

  • MediaQuery – When the widget should be fullscreen
  • LayoutBuilder – When the widget is nested and has parent constraints
  • Both – Sometimes you need both for complex layouts
Layout Strategies

1. Flexible Layouts

Use Flexible , Expanded , and Flex to create layouts that adapt to available space.

Flexible Layout Example Row( children: [ Flexible( flex: 1 , child: Container(color: Colors.blue, height: 100 ), ), Flexible( flex: 2 , child: Container(color: Colors.green, height: 100 ), ), Flexible( flex: 1 , child: Container(color: Colors.red, height: 100 ), ), ], )

2. Wrap Widget

Wrap automatically moves items to the next line when they don't fit.

Wrap Example Wrap( spacing: 8 , runSpacing: 8 , children: [ Chip(label: Text( 'Flutter' )), Chip(label: Text( 'Dart' )), Chip(label: Text( 'Responsive' )), Chip(label: Text( 'Design' )), // Items wrap to next line automatically ], )

3. GridView for Responsive Grids

Responsive Grid GridView.builder( gridDelegate: SliverGridDelegateWithMaxCrossAxisExtent( maxCrossAxisExtent: 200 , // Auto-adjusts columns mainAxisSpacing: 8 , crossAxisSpacing: 8 , childAspectRatio: 1.0 , ), itemCount: 20 , itemBuilder: (context, index) { return Card(child: Center(child: Text( '$index' ))); }, )

4. Orientation Builder

Orientation-Aware Layout class OrientationAwareWidget extends StatelessWidget { @override Widget build (BuildContext context) { final orientation = MediaQuery. orientationOf (context); if (orientation == Orientation.portrait) { return PortraitLayout(); } else { return LandscapeLayout(); } } }
Adaptive Navigation

Navigation is one of the most important adaptive UI elements. Switch between NavigationBar (mobile) and NavigationRail (tablet/desktop).

Adaptive Navigation Example class AdaptiveNavigation extends StatefulWidget { @override _AdaptiveNavigationState createState () => _AdaptiveNavigationState(); } class _AdaptiveNavigationState extends State<AdaptiveNavigation> { int _selectedIndex = 0 ; final List<Widget> _pages = [ HomePage(), SearchPage(), ProfilePage(), ]; final List<NavigationDestination> _destinations = [ NavigationDestination(icon: Icon(Icons.home), label: 'Home' ), NavigationDestination(icon: Icon(Icons.search), label: 'Search' ), NavigationDestination(icon: Icon(Icons.person), label: 'Profile' ), ]; @override Widget build (BuildContext context) { final width = MediaQuery. sizeOf (context).width; return Scaffold( body: Row( children: [ // Adaptive navigation if (width >= 600 ) NavigationRail( selectedIndex: _selectedIndex, onDestinationSelected: (index) { setState (() => _selectedIndex = index); }, labelType: NavigationRailLabelType.all, destinations: _destinations, ), Expanded( child: _pages[_selectedIndex], ), ], ), // Bottom navigation bar for mobile bottomNavigationBar: width < 600 ? NavigationBar( selectedIndex: _selectedIndex, onDestinationSelected: (index) { setState (() => _selectedIndex = index); }, destinations: _destinations, ) : null , ); } }
Adaptive Dialogs

Dialogs should adapt to screen size — fullscreen on mobile, modal on desktop.

Adaptive Dialog void showAdaptiveDialog (BuildContext context) { final width = MediaQuery. sizeOf (context).width; final isMobile = width < 600 ; if (isMobile) { // Fullscreen dialog on mobile showModalBottomSheet( context: context, isScrollControlled: true , builder: (context) => MyDialogContent(), ); } else { // Modal dialog on larger screens showDialog( context: context, builder: (context) => AlertDialog( title: Text( 'Title' ), content: MyDialogContent(), actions: [ TextButton( onPressed: () => Navigator. pop (context), child: Text( 'Cancel' ), ), ], ), ); } }
Step-by-Step Implementation

Follow these steps to make your app responsive:

1.
Define breakpoints – Choose widths for mobile, tablet, and desktop
2.
Measure screen size – Use MediaQuery or LayoutBuilder
3.
Create responsive layouts – Use Flexible , Expanded , Wrap
4.
Adapt navigation – Switch between NavigationBar and NavigationRail
5.
Adapt dialogs – Fullscreen on mobile, modal on desktop
6.
Test all sizes – Use device preview and resize your browser
7.
Polish and refine – Adjust spacing, fonts, and interactions for each size
Common Mistakes
❌ Mistake 1: Using hardcoded pixel sizes

Hardcoding sizes like width: 300 doesn't adapt to different screen sizes.

✅ Correct: Use relative sizes

Use MediaQuery , FractionallySizedBox , or Expanded for flexible sizes.

❌ Mistake 2: Only testing on one device

Testing only on your phone means you miss tablet and desktop issues.

✅ Correct: Test multiple sizes

Use Flutter's device preview or resize your browser to test different screen sizes.

❌ Mistake 3: Using MediaQuery inside build too often

Calling MediaQuery.sizeOf(context) triggers rebuilds when the screen size changes.

✅ Correct: Cache sizes when needed

Store the screen size in a variable or use LayoutBuilder for local constraints.

❌ Mistake 4: Forgetting about foldables

Foldable devices have unique layouts (dual screen, hinge). Test with foldable emulators.

✅ Correct: Support foldable devices

Use DisplayFeature and Foldable APIs for foldable support.

🎯 Key Takeaway

Responsive design is about fitting your UI into the available space, while adaptive design is about making it usable in that space. Use breakpoints to determine when to switch layouts. Measure screen size with MediaQuery (for full screen) or LayoutBuilder (for local constraints). Use flexible layouts , adaptive navigation , and responsive grids to create apps that look great on any device.
